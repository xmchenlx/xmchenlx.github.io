下载方法

安装依赖

使用npm、yarn等工具 npm install file-saver -s安装saveAs的依赖。

在导出功能的页面里，引入依赖。

import FileSaver from "file-saver";

编写逻辑

单击导出/下载按钮时，通过axios向服务器发送请求后，服务器回应的response里应该是一个文件流。我们在Promise里的.then()方法将文件流用Blob构造函数进行处理；

至于文件名，文件流是没有文件名信息的。后端会把文件名信息处理成UTF-8的代码后放到请求的Response Headers里，前端在得到回复后，另行去处理这段信息，最后，再用SaveAs依赖命名文件名后浏览器即可自动通过下载工具进行下载。

P.S:如果在自己封装的axios做了统一的请求状态拦截判断，需要用额外的办法让这种下载文件类型的请求跳过状态判断拦截，在自己的函数里。因为返回的文件流没有请求状态，但异常/失败的情况有。而且因为需要取headers里的信息，不能像常规一样为了方便只返回data本身而应该返回完整的请求结果信息。
